## The Main Problem

The ongoing challenge for many development teams is the significant effort required to create and maintain high-quality documentation. Developers often find themselves repeatedly building documentation sites from scratch for each new project, or contending with overly opinionated frameworks that lack flexibility. This inevitably leads to inconsistent branding, suboptimal performance, and a frustrating authoring experience.

## What is Fumadocs?

Before looking at Unmint, we must understand the foundation it is built upon: **Fumadocs**. 

Fumadocs is a "headless" documentation framework. This means it provides developers with the lower-level React/Next.js components, file routing logic, and utilities needed to build a documentation site from scratch. It directly addresses common pain points by offering a cohesive, open-source framework that eliminates repetitive setup. 

Unlike alternatives that sacrifice flexibility for ease of use, Fumadocs strikes a perfect balance. It streamlines content creation via Markdown and MDX support and efficiently transforms that content into usable data within a Next.js environment. 

However, because it is headless, the hidden cost of Fumadocs is **engineering time**. Setting it up, customising the layout, and handling ongoing maintenance can easily add up to several days of initial work.

## Unmint, the younger brother.

Unmint takes those exact Fumadocs building blocks and packages them into a ready-to-use layout. Instead of writing complex code to connect everything yourself, Unmint provides a pre-styled, "out-of-the-box" template powered entirely by a single configuration file. This means you can style your whole website without touching a single line of layout code.

That is not to say customisation isn't an option. Within Unmint's fixed layout, you use that single configuration file to completely control:
* **Branding:** Effortlessly swap out logos, favicons, and accent colours.
* **Navigation:** Customise header links, social icons, top tabs, and action buttons.
* **Structure:** Define the sidebar layout, page order, titles, and icons.

Within the pages themselves, you still have total freedom to craft content using built-in MDX components like tabs, accordion dropdowns, and warning callouts (or even create your own). 

Ultimately, Unmint significantly cuts down the engineering costs associated with raw Fumadocs. By intentionally trading off absolute layout customisability, it delivers a highly professional, high-performance documentation site in a fraction of the time.
